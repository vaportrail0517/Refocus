# Refocus

Refocus は、「気づいたら SNS や動画で長時間経っていた」を防ぎ、本当にやりたいことに意識を戻すためのタイマー付きナッジアプリです。

対象のアプリを使っている間だけ画面上に連続使用時間をオーバーレイ表示し、使いすぎているタイミングで、ユーザーが登録した「やりたいこと」や休憩への切り替えを提案します。

強制ブロックではなく、「気づき」と「選びやすい選択肢」で行動をサポートすることを目的としています。

---

## 特徴

- 連続使用時間の可視化
  対象アプリの前面利用中だけ、小さなタイマーが画面上に表示されます。

- 穏やかなナッジ
  一定時間以上の連続使用や時間帯をトリガに、「やめどき」「切り替えどき」を提案します。強制的なブロックは行いません。

- 「やりたいこと」への橋渡し
  趣味・休憩など、ユーザー自身が登録した行動候補を提案として表示します。

- プライバシ重視
  使用状況データは端末内で完結し、外部サーバには送信しない設計を前提としています。

---

## 対応プラットフォーム

- 現在: モバイルアプリ（Android クライアント）
- 将来: iOS クライアント（計画中）

このリポジトリは現時点では **Android 版 Refocus のみ** を含みます。
将来の iOS 版は、同じコンセプトと体験を目指しつつも、基本的には別実装として開発する予定です。

---

## アーキテクチャ概要

Refocus（Android 版）は、概ね次のレイヤに分かれた構成を想定しています。

- ドメイン層
  セッション、設定、提案ロジックなど、「アプリとして何をしたいか」を表現するロジック。

- データ層
  利用履歴や設定、「やりたいこと」などを端末内ストレージ（DB / Key-Value）に保存・読み出しする部分。

- システム層
  OS が提供する前面アプリの検知、オーバーレイ表示、起動時フック、各種権限の扱いなどを担う部分。

- プレゼンテーション層
  画面 UI、オーバーレイ UI、設定・オンボーディング・履歴・統計など、ユーザーとのインタラクションを担当する部分。

## 開発者向け（Android 概要）

### 必要環境

- Android Studio（最新版を推奨）
- JDK / Android SDK（通常の Android アプリ開発環境）

### ビルド / 実行

1. このリポジトリをクローンします。
2. Android Studio でプロジェクトを開きます。
3. 対応デバイスまたはエミュレータを選択し、ビルド・実行します。

※ 詳細なセットアップ手順やレイヤ構成の説明は、必要に応じて別ドキュメントとして追加予定です。
